Git y Github sincronizado via ssh

by esteban vergara git github linux 0 comments

Siempre sucede que cuando configuramos nuestro proyecto para que se sincronice con un repositorio de Github acabamos configurando el origin via https lo que nos obliga a poner nuestro nombre de usuario y contraseña cada vez que queremos hacer un "push"

Para evitar esto lo que tenemos que hacer es configurar el upstream via ssh en lugar de https Es bastante sencillo y una vez que lo hacemos seguro que lo haremos para todos nuestros proyectos. En primer lugar debemos de crear una clave rsa y posteriormente "decirle" a Github que nuestro servidor con esa clave es confiable. Y poco má, así que nos pongamos manos a la obra.

Lo primero es lo primero. O estamos en Linux o estamos wn WSL (Windows Subsytem for Linux). MacOs tiene comandos equivalentes aunque pueden tener alguna diferencia mínima. No recomendamos usar Windows para tareas de programación a menos que estéis programando aplicaciones para el entorno Windows.

Paso 1: Crear las claves RSA
Si ya tienes un juego de clave creadas puedes saltarte este paso. No obstante Github te pide un juego de claves diferentes para cada proyecto. Cuando ingresas la misma clave en otro de tus proyectos, se "queja" de que esa clave RSA ya está usadas. Para crear un nuevo juego de claves vamos a la linea de comandos y copiamos lo siguiente:

ssh-keygen -o -t rsa -C "your@email.com"

Asegurandonos de cambiar "your@email.com" por TU email.

Paso 2: Inicializar GIT
Ahora vamos al directorio del proyecto y vamos a inicializar Git. Otra vez, si ya has inicializado Git no vuelvas a hacerlo porque machacaras todo o te dará error. Si aun no has hecho nada dentro del directorio del proyecto escribe lo siguiente:

git init

Paso 3: Crear el repositorio en Github
Entramos a Github y nos logamos. Una vez logados damos al botón de crear un nuevo proyecto y rellenamos todos los datos. Asegurate de no crear el proyecto con NADA. No agregarle ni README, ni Licencia, ni .gitignore

Paso 4: Cargar la clave en Github
Volvemos a nuetro ordenador y en la linea de comando ponemos lo siguiente:

cat ~/.ssh/id_rsa.pub

El contenido de nuestra clave saldrá por pantalla. Lo seleccionamos todo (nuestro email es con lo que acaba la clave) y copiamos su contenido.
Ahora volvemos a Github y vamos a la opció de "Settings" y luego en el menú de la izquierda seleccionamos "Deploy Key". Ahora seleccionamos "Add Key" y copiamos el contenido de la clave en el recuadro que nos ha habilitado.

Paso 5: Creación de ficheros base
Volvemos a nuetro ordenador y desde la linea de comandos creamos los ficheros de README, Licencia y .gitignore. Todos estos los podemos copiar de los estándar de Github o de otro proyecto que ya tengamos.

Paso 6: Sincronización final
Vamos a hacer la primera sincronización con los ficheros creados hace un momento y terminar las ultimas configuraciones.
Para esto escribimos lo siguiente en nuestra linea de comandos:

git add .



Con esto agregamos los ficheros al Staging (si no sabes qué es el staging area tienes que coger un curso de git)

git commit –m "Inicializacion de repo"



Confirmamos los cambios del staging. Puedes poner el comentario que prefieras.

git remote add origin git@github.com:usuario/repositorio.git



Aquí ve con cuidado. No es solamente copiar y pegar. Tienes que reemplazar "usuario" por tu usuario de Github y "respositorio.git" por tu nombre de repositorio. La extensión git se usa para las conexiones via ssh.

git push -u origin master



Y ya por último empujamos los cambios desde nuestro repositorio local (master) hacia Github (origin). La opción "-u" se pone solamente la primera vez que haces el push para crear el "branch" o rama. Las siguientes veces puedes omitir el "-u"
